LCA介绍

和它的全称一样，lca（LowestCommonAncestorLowest Common AncestorLowestCommonAncestor）目的就是任意两个点最近的公共祖先在哪

性质 （或许你不需要记这个）

From OI-wiki

为了方便，我们记某点集 S={v1,v2,...,vn}S = \left\{ v_1,v_2,...,v_n \right\}S={v1​,v2​,...,vn​} 的最近公共祖先为 LCA(v1,v2,...,vn)LCA(v_1,v_2,...,v_n)LCA(v1​,v2​,...,vn​) 或 LCA(S)LCA(S)LCA(S)

  1. LCA({u})=uLCA(\left\{u\right\}) = uLCA({u})=u ;

  2. 若 uuu 是 vvv 的祖先，当且仅当 LCA(u,v)=uLCA(u, v) = uLCA(u,v)=u ;

  3. 如果 uuu 不为 vvv 的祖先，并且 vvv 不为 uuu 的祖先，那么 u,vu, vu,v 分别处于 LCA(u,v)LCA(u, v)LCA(u,v) 的两颗不同子树中 ;

  4. 前序遍历中， LCA(S)LCA(S)LCA(S) 出现在所有 SSS 元素之前，后续遍历中 LCA(S)LCA(S)LCA(S) 则出现在所有 SSS 元素之后 ;

  5. 两点集并的最近公共祖先为两点集分别的最近公共祖先，即
    LCA(A∪B)=LCA(LCA(A),LCA(B))\qquad LCA(A \cup B) = LCA(LCA(A), LCA(B))LCA(A∪B)=LCA(LCA(A),LCA(B)) ;

朴素求法

朴素求LCA的想法很简单，只要一层层往上走直到两个点相同即可（但是有很多小细节需要注意）。

​ 1.计算每个点的父节点以及每个点的深度（到父节点的距离）显然，我们需要定义dep[N], root[N]两 个数组来分别存储 iii 点的深度和 iii 点的父节点，在求LCA需要先跑一遍dfs。

​ 2.接下来就是LCA函数了，需要注意的是，对于 LCA(x,y)LCA(x, y)LCA(x,y) 其中 xxx 的深度必须和 yyy 相同才能一起往 上走不然就会~~（爱人错过），所以在网上走前需要把 xxx 和 yyy 的深度弄到相同。

倍增求法

和朴素的求法不同，倍增求法用st表进行预处理，可以减少向上走的次数，从而减少复杂度。

因为需要减少向上走的次数，对于 st [i] [j] 存储的是从i点出发，走了(1<<j)个点到的位置。

​ 1.首先我们需要预处理。（dfs还是要跑的，但是在此处省略，和朴素LCA一样）

​ 2.接下来写LCA函数：这里需要对两个部分分别修改。

​ 2.1对于让 xxx 和 yyy 深度相同的部分，我们需要用到一些2进制的知识

​ 对于 xxx ，需要向上走到步数是 xxx 和 yyy 的深度差。（向上走一步，深度-1）

​ 所以记录下 lenlenlen 为 xxx 和 yyy 的深度差。

​ 我们需要知道的是，lenlenlen 如何分解成 222 的次方相加（因为st表是以2的次方存储的）。

​ 举个栗子

​ 假设 len=100len = 100len=100，转成二进制是 110010011001001100100 ，显然，

​ 只有对应位是1的位置才对总体的值有贡献，

​ 所以我们当位置存在 111 时，向上走(1<<i)个点就行了（i指对应的位数）

​ 这样，总共走的点数就会是 lenlenlen 。

​ 2.2接下来需要 xxx 和 yyy 一起向上走。

​ 这里用到二分的思想，但是写法和常规二分不太一样。

​ 如果 xxx 和 yyy 一起跳了(1<<i)个点后，x == y了，说明跳过了

​ 它们最近的公共祖先深度必然<=当前点的深度。

​ 如图，对于5和4号点，显然跳过了它们的最近公共祖先

​ (顺手推荐下很方面画图的网站https://csacademy.com/app/graph_editor/)

​ 所以显然，我们需要在第一个 x’ != y’ 的地方更新x和y才能保证不走过。

​ (x’ 和 y’指在循环中走跳(1<<i)个点时到的点)

​ 这样得到的点应该是在 xxx 和 yyy 的最近公共祖先的前一个点，所以需要返回root[x]。
